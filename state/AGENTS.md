# state/AGENTS.md

Scope: current project state files under `state/`.

State file rules:

- State files describe current project state, handoff context and near-term focus; they are not stable architecture contracts.
- Update `progress.md` and `handoff.md` after meaningful work.
- `current-focus.md` must reflect only the next allowed task and must not broaden scope beyond the roadmap/prompt.
- Do not put secrets, credentials, real client data or private notes in state files.
- Do not duplicate large sections from stable docs; link to docs instead.
- Keep state concise enough for the next Codex session to understand what changed and what to do next.
